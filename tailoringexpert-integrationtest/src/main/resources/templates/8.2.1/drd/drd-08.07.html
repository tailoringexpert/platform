DRD-08.07 - Static Verification Reporting
Reference:
Purpose: To demonstrate that the S/W code implements correct control and data flow, and the absence of run-time errors.
Content: The report shall include that:
  • the following run-time errors are absent:
  • square roots of a negative number,
  • overflows and underflows,
  • out-of-bound array and pointer accesses,
  • illegal type conversion,
  • reading of non-initialized data;
  • non-deterministic behavior is absent;
  • data corruption is absent;
  • security breaches is absent;
  • internal consistency between S/W units is checked (number and types of parameters passed to functions - including function pointers in C language);
  • the code implements correct control and data flow.
Format: The required content will be automatically produced by the tool PolySpace Verifier.
