DRD-08.03 - Software [Verification / Acceptance Test] Plan
Reference: ECSS-E-ST-40C Space engineering - Software
ECSS-Q-ST-80C Space product assurance - Software product assurance
Purpose: The Software [Verification / Acceptance Test] Plan describes the approach and the organization aspects to implement the software verification / acceptance activities.
Content:

Title

The Software [Verification / Acceptance Test] Plan to be created on this DRD shall be titled as one of the following:
  • [insert a descriptive modifier] Software Verification Plan,
  • [insert a descriptive modifier] Software Acceptance Test Plan
The descriptive modifier shall be selected to clearly identify the applicable product and level specification.

Title page

The title page shall identify the project document identification number, title of the document, date of release and release authority.

Contents list

The content list shall identify the title and location of every clause and major subclause, figure, table and annex contained in the document.

Foreword

A foreword shall be included in the document which describes as many of the following items as are appropriate:
  • identification of which organizational entity prepared the document,
  • information regarding the approval of the document,
  • identification of other organizations that contributed to the preparation of the document,
  • a statement of effectivity identifying which other documents are cancelled and replaced in whole or in part,
  • a statement of significant technical differences between this document and any previous document,
  • the relationship of the document to other standards or documents.

Introduction

An introduction may be included to provide specific information or commentary about the technical contents.

1 Scope and applicability

This clause shall describe the scope, applicability and purpose of the Software [Verification / Acceptance Test] Plan.

1.1 Scope

This subclause shall contain a statement to define the goal of the Software [Verification / Acceptance Test] Plan.

1.2 Applicability

  1. This subclause shall contain a statement to define the use of the Software [Verification / Acceptance Test] Plan
  2. This subclause shall briefly define the purpose of this Software [Verification / Acceptance Test] Plan:
    • identify the software project (and the actual software items) for which this Software [Verification / Acceptance Test] Plan is written;
    • identify the software products (documentation and code) to be verified and the documentation against which these will be verified;
    • outline the goals of the verification / acceptance activities;
    • specify the intended readers of this Software [Verification / Acceptance Test] Plan.

1.3 Document organization

This subclause shall describe what the rest of the document contains and explain how the rest of the document is organized.

4 Software verification / acceptance process overview

This clause shall describe the approach to be utilized to implement the verification / acceptance process throughout the software life cycle, the required verification / acceptance effort, and the level of required independence for the verification / acceptance tasks.

4.1 Organization

  1. This subclause shall describe the organization of the documentation review, proofs, tracing activities.
  2. Topics that shall be included are:
    • roles,
    • reporting channels;
    • levels of authority for resolving problems;
    • organization relationships;
    • level of required and implemented independence.

4.2 Master schedule

  1. This subclause shall identify the schedule for the planned verification / acceptance activities.
  2. Information about reviews shall also be included.

4.3 Resource summary

This subclause shall summarize the resources needed to perform the verification / acceptance activities such as staff, hardware and software tools.

4.4 Responsibilities

This subclause shall define the specific responsibilities.

4.5 Tools, techniques and methods

This subclause shall identify the software tools, techniques and methods used to execute the verification / acceptance tasks throughout the software life cycle.

5 Control procedures for verification / acceptance process

This clause shall contain information (or reference to) about applicable management procedures concerning the following aspects:
  • problem reporting and resolution;
  • deviation/waiver policy;
  • control procedures

6 Verification / acceptance activities

  1. For each of the tasks pertaining to the verification / acceptance process, this clause shall describe which methods, tools, facilities are utilized and which outputs have to be produced. This includes the identification of internal reviews, walkthrough, inspections, etc.
  2. When the need of utilization of specific methods and tools is identified (e.g. formal proof), it shall be specified.
  3. This clause shall be structured in the following subclauses and each one should contain the above specified information per each software item under verification / acceptance.

7 Software verification / acceptance reporting

  1. This clause shall describe how the results of implementing then will be documented reports produced by the implementation of the verification / acceptance process shall be identified.

8. Software quality requirements verification (as per ECSS-Q-ST-80 clause 6.2.6.1)
