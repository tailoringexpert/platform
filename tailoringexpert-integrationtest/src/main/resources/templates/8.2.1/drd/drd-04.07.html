DRD-04.07 - Functional Tree (FT)
Reference: MIL-HDBK-338 Electronic Reliability Design Handbook
ECSS-Q-ST-10C Space engineering - System engineering general requirements
ECSS-Q-ST-30-02C Space product assurance - Failure Modes, effects (and criticality) analysis (FMEA/FMECA)
Purpose: In order to design, develop and prove a system/subsystem, the mission and consequent functions, that the system shall perform, shall be clearly established. A FT shall be constructed to depict the functional elements of the analysed product and their interdependencies.
Content: Tree structures shall provide clear visibility of the large number of functional elements making up a system. A tree structure shall enable errors, omissions, inconsistencies and duplications to be detected through the branches. A hierarchical structure starting at system level working down in detail shall allow verification that the lower level functions are consistent with the top-level functions. The functional elements are shown as blocks. A numerical reference for all blocks in the functional tree shall be provided by using a standard coding system. The functional interdependencies between the blocks are shown as logic gates.
The following functional analysis steps shall be implemented:
  • define the system and boundaries of the system under analysis by using the relevant requirements documents;
  • define the level of detail (depth of analysis);
  • identify the system functions/operational modes/operational scenarios;
  • represent the system by preparing the function tree(s), or functional matrix(ces), or functional block diagram(s).
Format: Similar to the following example. The system functionality shall be distributed throughout the different design levels (e.g. systems, subsystems, units).
